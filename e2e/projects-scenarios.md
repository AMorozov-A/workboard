# E2E: сценарии проектов (Freelance CRM)

Краткая фиксация сценариев для Playwright (`e2e/projects.spec.ts`). Локаль по умолчанию — русская.

## 1. Создание проекта через UI

- **Предусловие:** `registerAndLogin` (токен в `localStorage`, `/app/projects`).
- **Результат:** проект с заданным именем виден в списке (`projects-page-root`).

## 2. Редактирование проекта

- **Предусловие:** пользователь залогинен, в списке есть проект (создание через UI в том же тесте).
- **Результат:** новое название отображается в таблице.

## 3. Удаление с подтверждением

- **Предусловие:** в списке есть проект.
- **Результат:** после confirm «Удалить этот проект?» проект исчезает; при отсутствии проектов — пустое состояние «Пока нет ни одного проекта».

## 3b. Отмена удаления (опционально)

- **Результат:** по «Отмена» проект остаётся в списке.

## 4. Переход на карточку проекта

- **Предусловие:** в списке есть проект.
- **Шаги:** клик по строке таблицы (не по кнопкам действий).
- **Результат:** URL `/app/projects/<ключ>`; видна вкладка «Задачи», секция «Задачи проекта».

## 5. Негатив: валидация без названия

- **Результат:** при пустом названии и заполненном клиенте кнопка «Создать» неактивна (или показ «Введите название проекта»).

## 6. Негатив: ошибка сервера при создании

- **Предусловие:** перехват `POST **/api/v1/projects` → ответ 500 (`stubProjectsCreateFailure`).
- **Результат:** уведомление «Не удалось создать проект».

## Хелперы

- `e2e/helpers/auth.ts` — `registerAndLogin`, `registerViaAPI`, `loginViaAPI`.
- `e2e/helpers/projects.ts` — `stubProjectsCreateFailure` (мок для сценария 6).
